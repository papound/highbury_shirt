# Highbury International — E-Commerce Website

เว็บไซต์ขายเสื้อเชิ้ตชาย/หญิงสำเร็จรูป สำหรับแบรนด์ **Highbury International**

## Tech Stack

- **Next.js 16** (App Router, Turbopack)
- **TypeScript** + **Tailwind CSS v4**
- **Prisma 7** + SQLite (dev) / PostgreSQL 16 (prod)
- **NextAuth v5** (JWT)
- **shadcn/ui** (base-nova style)
- **UploadThing** — อัปโหลดรูปสินค้าและสลิปโอนเงิน
- **Recharts**, **Tiptap**, **ExcelJS**, **PromptPay QR**
- **react-markdown** — render เนื้อหาบทความ
- **Docker + Docker Compose** — production stack

---

## Development

### ข้อกำหนด

- Node.js **v22** (ใช้ nvm)

### ติดตั้งและรันโปรเจค

```bash
# ใช้ Node.js v22
source ~/.nvm/nvm.sh && nvm use 22

# ติดตั้ง dependencies
npm install

# สร้างและ seed ฐานข้อมูล (SQLite)
npx prisma migrate dev
npx prisma db seed

# รัน development server
npm run dev
```

เปิด [http://localhost:3000](http://localhost:3000)

## Production (Docker Stack)

### ติดตั้งบน Proxmox LXC (Debian 12)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/papound/highbury_shirt/main/scripts/install/highbury-shirt-docker.sh)
```

หรือผ่าน Proxmox helper script:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/papound/highbury_shirt/main/scripts/ct/highbury-shirt-docker.sh)
```

### Setup หลังติดตั้ง

```bash
# 1. สร้าง warehouse ใน /admin/settings ก่อน import สินค้า

# 2. Push schema และ seed ข้อมูลเริ่มต้น
docker exec highbury-app npx prisma db push
docker exec highbury-app npx prisma db seed

# 3. ดู logs
cd /opt/highbury-shirt && docker compose logs -f app
```

### อัปเดต application

```bash
cd /opt/highbury-shirt && git pull
docker compose build --progress=plain app
docker compose up -d
```

### Environment Variables (Production)

ไฟล์ `.env` ที่ `/opt/highbury-shirt/.env`:

```env
DB_USER=highbury
DB_NAME=highbury_db
DB_PASS=<auto-generated>
AUTH_SECRET=<auto-generated>
NEXTAUTH_URL=http://<SERVER_IP>
NEXT_PUBLIC_APP_URL=http://<SERVER_IP>
UPLOADTHING_TOKEN=CHANGE_ME
LINE_NOTIFY_TOKEN=CHANGE_ME
```

### Docker Services

| Service | Port | คำอธิบาย |
|---------|------|----------|
| `app` | 3000 | Next.js application |
| `db` | 5432 | PostgreSQL 16 |

เชื่อมต่อ PostgreSQL จากภายนอก:
```
Host: <SERVER_IP>:5432
```

## หมายเหตุสำหรับ Dev

- **Database**: ไฟล์ `dev.db` ถูก exclude จาก git — ต้อง run `npx prisma migrate dev && npx prisma db seed` หลัง clone
- **Production DB**: ใช้ PostgreSQL 16 ผ่าน Docker — schema patch อัตโนมัติใน Dockerfile (`sqlite` → `postgresql`)
- **Inventory**: ต้องสร้าง Warehouse ใน `/admin/settings` ก่อน import สินค้า ไม่งั้น inventory จะไม่ถูกสร้าง
- **Session**: JWT-based — หาก re-seed ฐานข้อมูล ต้อง sign out / sign in ใหม่เพื่อรับ user ID ที่ถูกต้อง
- **Uploaded files**: `public/uploads/` ถูก exclude จาก git
- **Blog**: ข้อมูลบทความอยู่ใน `lib/mock-blog.ts` (static) — เชื่อมกับ `BlogPost` model ใน DB สำหรับ production
- **Cart**: จำนวนสินค้าถูก clamp ตาม stock จริง ทั้ง Zustand store และ UI (`add-to-cart-button.tsx`)
